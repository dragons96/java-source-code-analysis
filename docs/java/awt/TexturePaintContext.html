    blend(int rgbs[], int xmul, int ymul): int public 是 否 否 否 否 否 Blends the four ARGB values in the rgbs array using the factors described by xmul and ymul in the following ratio: rgbs[0] * (1-xmul) * (1-ymul) + rgbs[1] * ( xmul) * (1-ymul) + rgbs[2] * (1-xmul) * ( ymul) + rgbs[3] * ( xmul) * ( ymul) xmul and ymul are integer values in the half-open range [0, 2^31) where 0 == 0.0 and 2^31 == 1.0. Note that since the range is half-open, the values are always logically less than 1.0. This makes sense because while choosing pixels to blend, when the error values reach 1.0 we move to the next pixel and reset them to 0.0.
