    UPPER_DOSTIME_BOUND: long 128L * 365 * 24 * 60 * 60 * 1000 private 是 是 否 否 否 Approximately 128 years, in milliseconds (ignoring leap years etc). This establish an approximate high-bound value for DOS times in milliseconds since epoch, used to enable an efficient but sufficient bounds check to avoid generating extended last modified time entries. Calculating the exact number is locale dependent, would require loading TimeZone data eagerly, and would make little practical sense. Since DOS times theoretically go to 2107 - with compatibility not guaranteed after 2099 - setting this to a time that is before but near 2099 should be sufficient.
